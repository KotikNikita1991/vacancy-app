# vacancy-app

Проект **«Вакансия»**: дашборд вакансий, аналитика, чек-лист оценки кандидатов, пользователи (админ). Бэкенд по умолчанию — веб-приложение Google Apps Script (POST JSON `{ "action": "...", ... }`).

## Что изменилось относительно одного `index.html`

- Разнесённые файлы: `css/app.css`, `js/config.js`, `util.js`, `api.js`, `data.js`, `app.js`.
- Запросы к API: таймаут, разбор не-JSON ответов, коды HTTP.
- Меньше XSS-риска: `escapeHtml` для пользовательских строк, открытие карточки вакансии по `data-vacid`, без `JSON.stringify` в `onclick`.
- Экран входа: сохранение URL API в `localStorage` (блок «Подключение к API» доступен только в `?debug=1`).
- Дашборд: сохранение периода и фильтров в `localStorage`, кнопка **CSV**.
- Аналитика: планы рекрутеров сохраняются в `localStorage`.
- Пользователи: отдельные действия `activateUser` / `deactivateUser` (если скрипт их не поддерживает — локально обновится список до следующей загрузки).
- Закрытие модалок по **Esc**.

## Запуск

Для корректной работы `fetch` откройте приложение через **локальный статический сервер**, например:

```bash
python -m http.server 5173
```

Затем в браузере откройте `http://127.0.0.1:5173/` (или тот порт, который указан в команде).

**Вход:** только через API (`action: "login"`).

**URL API** задаётся на экране входа в блоке «Подключение к API» (показывается в `?debug=1`) и сохраняется в `localStorage`.

Исходная вёрстка и логика перенесены из одностраничного `index.html`; стили — в `css/app.css`, скрипты — в `js/`.

**Cursor:** чтобы ИИ мог править файлы сам, в чате выберите режим **Agent** (не Ask). В Composer обычно есть переключатель режима рядом с полем ввода.

**Git** на вашей системе может быть не установлен — это не мешает запуску; при необходимости установите Git и выполните `git init` в папке проекта.
